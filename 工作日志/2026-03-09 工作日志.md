
## 2026-03-09 工作日志

1. **客群分析列表交互与排序**  
   **问题**：客群分析列表未按已分析优先排序，且点击“一键分析”会刷新页面，影响继续浏览群聊。  
   **处理**：调整群聊排序规则为已分析优先并加入手动刷新入口；将“一键分析”改为异步提交以避免页面刷新。  
   **结果**：已分析群聊排在前列，分析后页面不刷新，仍可继续浏览；可手动刷新列表。

2. **分析日志实时刷新异常**  
   **问题**：分析改为异步提交后日志不再实时刷新。  
   **处理**：状态接口支持按 task_id 查询；前端持久化当前任务 ID 并轮询时携带。  
   **结果**：日志与状态恢复实时更新。

3. **全量分析报错与抓取上限控制**  
   **问题**：全量分析出现 `'NoneType' object has no attribute 'get'`，且数据量过大需要限制。  
   **处理**：为上游消息的 `msgData/fromInfo` 增加空值防御；将抓取上限改为可配置并默认 2000。  
   **结果**：异常消除；单次分析最高抓取 2000 条，后续可增量继续。

4. **新增闭环问题模块与拉取鉴权**  
   **问题**：缺少闭环问题独立模块；拉取接口鉴权与错误反馈不清晰。  
   **处理**：新增闭环问题页面与数据落库、拉取与同步入口；修正鉴权方式并增加错误提示与展示。  
   **结果**：闭环问题模块可拉取与展示，错误信息可定位。

5. **闭环写入知识库策略调整**  
   **问题**：闭环数据写入到了原 QA 知识库，且一条问题生成一个文档。  
   **处理**：改为写入独立闭环知识库数据集；按客户合并多条问题写入同一文档，并维护客户级文档映射。  
   **结果**：闭环知识库与 QA 知识库分离；同一客户数据统一管理。

6. **闭环知识库写入参数不匹配**  
   **问题**：Dify 返回 `doc_form is different from the dataset doc_form` 导致写入失败。  
   **处理**：支持闭环模块独立配置 `doc_form` 与 `process_mode`；默认 doc_form 修正为 `text_model`，并暴露配置入口。  
   **结果**：参数可与数据集配置对齐，写入路径明确。

7. **分析页误触发接管与 Bot 接管失效**  
   **问题**：分析页一键分析完成后自动接管；修复后 Bot 界面接管也失效。  
   **处理**：将分析流程接管逻辑改为可开关，仅 Bot 接管流程启用。  
   **结果**：分析页仅分析不接管；Bot 界面接管恢复。

8. **QA 知识库批量拒绝失败**  
   **问题**：批量拒绝报唯一约束冲突（reviews.message_id）。  
   **处理**：批量拒绝前查询 Review，存在则更新，不再重复插入。  
   **结果**：批量拒绝不再触发唯一约束错误。

## 待办 / 后续计划

## 风险与注意事项
- 若闭环知识库数据集配置与页面参数不一致，仍会导致写入失败。
- 增量分析依赖 `last_ts` 断点；上游时间戳异常可能影响增量覆盖。

## 决策记录
- 闭环问题写入按客户合并为单文档，不再一条问题一个文档。
- 分析页一键分析不触发接管，接管仅由 Bot 界面发起。
